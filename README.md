## üèóÔ∏è Architecture

StarterKit follows **Clean Architecture** principles with clear separation of concerns:

```
üìÅ lib/
‚îú‚îÄ‚îÄ üìÅ src/
‚îÇ   ‚îú‚îÄ‚îÄ üìÅ core/                    # Core functionality
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ config/              # App configuration
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ design_system/       # UI components & themes
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ di/                  # Dependency injection
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ localization/        # Multi-language support
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ network/             # API client & networking
‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ üìÅ utils/               # Utility functions
‚îÇ   ‚îú‚îÄ‚îÄ üìÅ data/                    # Data layer
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ models/              # Data models
‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ üìÅ repositories/        # Data repositories
‚îÇ   ‚îú‚îÄ‚îÄ üìÅ domain/                  # Business logic layer
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ entities/            # Business entities
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ üìÅ repositories/        # Repository interfaces
‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ üìÅ usecases/            # Business use cases
‚îÇ   ‚îî‚îÄ‚îÄ üìÅ presentation/            # UI layer
‚îÇ       ‚îú‚îÄ‚îÄ üìÅ pages/               # Screen widgets
‚îÇ       ‚îî‚îÄ‚îÄ üìÅ widgets/             # Reusable UI components
```

## üõ†Ô∏è CLI Commands

StarterKit includes powerful CLI tools for rapid development:


Creates a complete project structure with:
- ‚úÖ Clean architecture folders
- ‚úÖ Configuration files
- ‚úÖ Asset directories
- ‚úÖ Example implementations

### üì± Generate Screen


Generates:
- üìÑ Screen widget
- üéØ State management
- üß™ Unit tests
- üìù Documentation

### üîå Generate API Service


Creates:
- üåê API client methods
- üìä Data models
- üîÑ Repository implementation
- üß™ Mock data for testing


- üèóÔ∏è Data class with JSON serialization

- üìù Documentation

## üìñ API Reference

### Core Classes

#### `AppConfig`
Configuration management for different environments.

```dart
class AppConfig {
  final String apiBaseUrl;
  final String appName;
  final String appVersion;
  final String buildNumber;
  final bool isDebug;
  
  // Factory constructors
  factory AppConfig.development();
  factory AppConfig.staging();
  factory AppConfig.production();
}
```

#### `ApiClient`
HTTP client for API communication.

```dart
class ApiClient {
  Future<ApiResponse<T>> get<T>(String endpoint);
  Future<ApiResponse<T>> post<T>(String endpoint, {dynamic data});
  Future<ApiResponse<T>> put<T>(String endpoint, {dynamic data});
  Future<ApiResponse<T>> delete<T>(String endpoint);
}
```

#### `ApiResponse<T>`
Generic response wrapper for API calls.

```dart
class ApiResponse<T> {
  final bool success;
  final String? message;
  final int? code;
  final T? data;
  final List<String>? errors;
  
  bool get isSuccess;
  bool get hasError;
  String? get errorMessage;
}
```
